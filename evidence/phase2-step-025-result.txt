  audit existed to catch the leak.

Fix shipped (4 parts):

  Part A — .claude/skills/screens/SKILL.md
    - §5 "Imagery convention" rewritten:
      Hero / content imagery: /placeholders/hero-{noun-phrase-kebab}.jpg
      Avatars: /placeholders/avatar-{first-name-lower}.jpg
      Case-study imagery: /placeholders/case-study-{client-slug}-{aspect}.jpg
      Generic content: /placeholders/content-{noun-phrase-kebab}.jpg
    - Hard prohibition on picsum.photos / images.unsplash.com /
      googleusercontent.com / gravatar.com etc.
    - Empirical motivator (test-app Mode B Run 2) inline

      LOCAL /placeholders/avatar-*.jpg + /placeholders/case-study-*.jpg
      paths for canonical avatars + case-study imagery

  Part B — .claude/skills/mockups/SKILL.md
    - Note added clarifying mockups MAY use external CDN URLs
      (legitimate for design-time visual review) but /screens REWRITES
      them downstream — mockup authors should NOT pre-emptively switch

  Part C — .claude/skills/agents/front-end/react-next/SKILL.md
    - New reviewer dimension under §Review:
      "performance — external CDN image URLs forbidden (bug-008)"

    - Empirical motivator inline + Playbook augmentation note

      a particular project

  Part D — scripts/audit-screens-external-cdn-urls.mjs (139 LOC)

      + docs/screens/**/*.html
    - 9 forbidden host patterns: picsum.photos, images.unsplash.com,
      unsplash.com, googleusercontent.com, gravatar.com,
      placeholder.com, placekitten.com, placebear.com, placedog.net,
      loremflickr.com, dummyimage.com

      summary on stderr (file path + line + host + truncated text)

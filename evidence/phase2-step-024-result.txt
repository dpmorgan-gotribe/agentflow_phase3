Empirical motivator:
  investigate-004 Q2 + test-app Mode B Run 2 (pipeline 15a61239) feat-home
  dispatches. tester-attempt-3.json output:

    "genuineProductBugs": [{
      "taskId": "home-screen",
      "builderAgent": "web-frontend-builder",
      "testFile": "apps/web/e2e/home.spec.ts",
      "failureMessage": "page.goto: Test timeout of 30000ms exceeded...",
      "likelyCause": "external CDN images block window.load"
    }]

  But the dispatches/feat-home/ directory contained NO
  web-frontend-builder-attempt-3.json. Builder retry-counter was at 1
  (max-3 → 2 remaining) but never got dispatched. Feature marked failed
  with abortReason "task home-e2e failed after 3 attempts" + the
  structured payload dropped on the floor.

Root cause (per investigate-004 Q2 findings):
  bug-121 routing block in feature-graph.ts:1849-1988 only inspects
  result.genuineProductBugs from the FIRST invokeAgent dispatch (line
  1376). When attempt-1 stalls → no parseable JSON → routing skipped →
  fall through to per-task retry loop (line 1991-2029). The retry loop
  is BLIND to retryResult.genuineProductBugs — it reads ONLY
  retryResult.taskStatus[t.id]:

    if (retryResult.taskStatus[t.id] === "completed") { ... break; }
    taskOutcomes[t.id] = "failed";
    result.errors[t.id] = retryResult.errors[t.id] ?? ...
    // retryResult.genuineProductBugs is NEVER READ

Same class as bug-007 + bug-109: routing surface schema field correctly
populated; orchestrator-side dispatch path missing.

Fix shipped:
  Inserted ~140 LOC routing block in feature-graph.ts at line ~2020
  (between retryResult.lastWritingAgent assignment and the
  taskStatus.completed check). Mirrors bug-121 first-dispatch algorithm:

    1. Group genuineProductBugs by originating task ID
    2. For each task: check builder retry budget; if available,
       increment counter, dispatch with HARD CONSTRAINT envelope
       inlining bug payload, commit on success
    3. After all builder dispatches: re-run tester ONCE for current task
    4. If tester verify passes → mark task completed, break while loop
    5. Otherwise → fall through to legacy failure path

  HARD CONSTRAINT envelope shape:
    "HARD CONSTRAINT — TESTER FLAGGED GENUINE PRODUCT BUG(S)
    Tester surfaced N bug(s) against your prior implementation
    (post-bug-009 retry-loop routing). You MUST apply these exact fixes:
    (1) <testFile>::<testName> — <failureMessage>
        likely cause: <likelyCause>
    ...
    Do not re-implement from the task spec — extend the existing
    implementation with the named changes."

  NEW SHAPE in bug-005: even the mechanical audit can silently no-op
  when its CONTRACT is DERIVED from an upstream artifact that doesn't
  model the contract surface. Triple-compounding bug:

    F1. Preview parser tag list (section|div|aside|footer|header|main|
        article) omits <a>/<button>/<span> — test-app preview's only
        dark-bg elements were on <a> tags (case-study cards).
    F2. Empty-vocab silently DISABLED D11 (guard: vocab.size > 0 → false
        → skip). Silent fail-OPEN on load-bearing check.
    F3. No bg-context tracking inside dark blocks → vocab pollution +
        screen-side over-counting.

  Real impact verified by investigate-003 bg-context-aware re-scan:
  2 of 12 test-app screens had hardcoded dark-text inside dark CTA
  bands — D11 passed them silently. Operator's eye was correct.

Four-part fix shipped at factory level (project-agnostic):

  Part A — scripts/audit-screen-pattern-consumption.mjs D11 strengthening
    A.1 — extend dark-block walker tag list with <a>/<button>/<span>;
          applies to BOTH preview parser AND screen-side scanner
    A.2 — fail-CLOSED on empty preview-dark-band vocab — emit structured
          stderr warning with worked-example template + push synthetic
          finding so reporter/strict paths fire consistently. --skip-D11
          opt-out flag for projects with no dark surfaces intentionally.
    A.3 — walkBgContext bg-context tokenizer maintains a stack as it
          walks; each tag's bg-context = (explicit dark class → dark) OR
          (explicit light class → light) OR (inherit from parent).
          Descendants of nested light pills no longer count as dark-band
          descendants. Eliminates F3 over-counting.
    A.4 — hardcoded dark-text-class blocklist as INDEPENDENT secondary
          check: ANY text-text-{primary,secondary,tertiary} /
          text-neutral-{700-950} / text-black inside dark-bg block
          (bg-context-aware) is ALWAYS a D11 finding. Defense in depth.
    Additional: family-level vocab matching (text-white/85 family
    accepts text-white/70 etc.) reduces noise without losing signal.
    Severity tiered: hardcoded findings = errors (always fail);
    outsideVocab findings = warnings (fail only with --strict).
    Per-screen findings clustered by dark-block start line for clean
    reporter output.

  Part B — scripts/audit-preview-coverage.mjs dark-band coverage assert
    - Preview MUST contain ≥1 element with dark-bg class AND ≥1
      descendant carrying a text-* class. Structural fix that
      upstream's the D11-vocab-derivation gap.
    - Exit 1 with worked-example HTML template on miss.

  Part C — .claude/skills/screens/SKILL.md §3.5.2 extension
